Тестування GET
1.Перевірка, що файл /200 існує в кеші
curl -i http://localhost:8080/200
2.Перевірити, що сервер повертає 404, якщо файлу немає.
curl -i http://localhost:8080/999
3.Перевірка, що якщо не ввести HTTP code.
curl -i http://localhost:8080/

Тестування PUT
1.Перевірка, чи сервер зможе прийняти файл і правильно зберегти його в кеш-папку (./test), повернувши статус 201
curl -i -X PUT --data-binary "@dummy-image.jpg" http://localhost:8080/201
2. Перевірка, чи можна отримати файл, який щойно створив.
curl -i http://localhost:8080/201

Тестування DELETE
1.Перевірка, чи можна видалити файл, який створив.
curl -i -X DELETE http://localhost:8080/201
2.Перевірка, чи поверне сервер 404, якщо спробувати видалити вже видалений або неіснуючий файл.
curl -i -X DELETE http://localhost:8080/201

Тестування 405
1.Перевірка, чи сервер блокує метод, який не реалізований, наприклад POST.
curl -i -X POST http://localhost:8080/200
